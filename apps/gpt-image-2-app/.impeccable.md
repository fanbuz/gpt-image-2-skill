# Design Context — gpt-image-2-app

> 这份文件是 impeccable 系列设计 skill 在本前端项目里的「初始上下文」。所有设计任务(craft / shape / animate / polish / harden / colorize 等)在动手之前都要先读这里。
>
> **位置约束**:本上下文只覆盖 `apps/gpt-image-2-app/`(Tauri 桌面前端)。CLI / Skill 包内的字符串不在视觉系统范畴。

## Users

**核心用户**:中文为主的 AI Agent 开发者、设计 / 视觉创作者、技术爱好者。
他们在以下两类场景里打开这个 App:

1. **创作 / 灵感时段**(主要场景)— 调 prompt、跑 Codex 或 OpenAI Image API、改参考图、刷 history、再调一次。会停留较长时间,愿意盯着结果反复观察。
2. **桌面快捷出图** — 中途切到 App 出一张 logo / icon / 预览图,关掉。要求是「打开 → 出图 → 关」一条直线。

**同时存在的隐含读者**:这是 `gpt-image-2-skill` 整套工具(CLI + Skill + App)里唯一的 GUI。它既是产品的「门面」,也是给中文社区演示「Codex / GPT Image 2 能做什么」的橱窗。设计水平本身就是产品 marketing 的一部分。

**使用语境**:macOS / Windows / Linux 桌面;长时间停留为主;深夜 / 弱光环境概率高;窗口经常并排其他工具(终端、设计稿、参考图)。

## Brand Personality

**3 词**:**Liquid · Luminous · Precise**

- **Liquid** — 表面有材质感、有流动感、不像扁平 Web。Glass、blur、subtle parallax 是有意为之的「材料语言」,不是装饰。
- **Luminous** — 整个界面有「内部光源」的感觉。紫 → 青 → 品红的品牌色像是从屏幕背后透出来的光,而不是涂在表面上的颜料。
- **Precise** — 工程级的下手精度。spacing 用 token、radii 用 token、字号用 scale。1px 错位都算回归。

**情绪目标**:用户每次打开应该感到「这个工具懂我在做什么,而且它比我更在意细节」。不是炫技,不是讨好,是工匠感。

**Tone of voice**(UI 文案):中文为主,简短、确定、不卖萌。错误信息说人话,不用「Oops」「哎呀」。空状态可以有少量诗意,但不饶舌。

## Aesthetic Direction

### 主轴:Liquid Glass(保留并精修)

当前的视觉系统已经成型,后续工作是**沿这条主轴精修与扩展**,不是 pivot。核心要素:

- **单一深色主题**(`--bg: #06060a`),light theme 已退役,不要尝试加回来。
- **Liquid glass 表面**:`surface-panel` 是核心容器材料 — translucent + backdrop-blur + 顶部 1px highlight。glass alpha 由用户在设置里可控(`--glass-alpha`)。
- **品牌渐变**:`violet #a78bfa → cyan #67e8f9 → magenta #f0abfc`。**只用于光晕、accent 边、状态高亮、icon、装饰光斑**,不做大面积色块,不做 gradient text(SKILL 绝对禁令)。
- **环境光**:`.desktop` 上左 violet 光晕 + 右下 cyan 光晕的 radial gradient,加 grid paper 底纹 — 这是「场」的暗示,不是装饰。新场景可以借用同一套语法,但避免叠加过多光源稀释了主场。
- **Density**:用户可在设置切 compact / comfortable,设计时**默认 comfortable**,但所有间距走 `--d-*` token,以保证 compact 模式不破。
- **Radii**:`6 / 10 / 14 / 18`(`--r-sm` → `--r-xl`)。表面级容器用 `xl`,inline 控件用 `md`,标签 / chip 用 `sm`。不要再引入新的圆角值。
- **Shadow**:已有 sm / md / lg / popover 四档,不再扩展。深色场景下阴影更多是「分层」,不是「立体」,克制使用。

### 参考(可借鉴的方向)

- Apple visionOS / macOS Sequoia 的玻璃材质语法(透明、折射、光的传递)
- Arc Browser 的可玩性与微妙惊喜(但收敛到工具语境)
- Linear 的精度感与排版克制
- Raycast 的密度与键盘优先(但视觉更松、更有光感)
- Figma FigJam / Framer 的「场感」(grid + ambient light + stickers)— 但只取「场」,不取 playful 元素

### 反例(明确不要)

- 通用 SaaS dashboard 的「卡片网格 + icon + heading + 三行文案」模板
- 冷冰冰的开发者工具感(纯黑 + 等宽 + 绿色高亮)
- Generic AI dark mode:cyan-on-black + 大面积紫蓝渐变 + 发光按钮(SKILL 明确反对)
- Glassmorphism 滥用 — glass 是有目的的分层工具,不是任何卡片都贴一层模糊
- 大量 emoji / 拟人化 / 卖萌
- gradient text、>1px 的 side-stripe border(SKILL 绝对禁令)

### 字体方向

**当前**:**General Sans** (Indian Type Foundry,免费商用,Fontshare CDN 加载) + JetBrains Mono + Source Serif 4。

General Sans 替换 Inter 是第二轮 impeccable audit 的产物。选它的原因:

- 数字 single-storey,字怀开阔,跟 hero 渐变 + tabular-nums chip 配合得更稳
- 平衡刀感与流畅感,贴合 Brand Personality 三词里的「Precise」,不至于变成另一支冷面 grotesque
- 不在 impeccable 的 reflex_fonts_to_reject 列表里,避开了 Inter / DM Sans / Plus Jakarta 这一批 SaaS 默认
- 免费商用,fallback 链 SF Pro / PingFang 在 Tauri 离线时仍是高质量

**Mono**:保留 JetBrains Mono(开发者熟悉感强,密度档够用),不和 General Sans 抢戏。
**Display**:暂未单独引入 display face,hero "GPT Image 2" 直接走 General Sans 600 + clamp 字号 + brand gradient on "Image"。
**CJK**:依然走系统 fallback(PingFang SC),General Sans 与 PingFang 的灰度匹配良好,没有「中文比西文重一档」的失衡。

**未来如果还要再换**:

- 任何替换都先按 SKILL 的 font_selection_procedure 重跑一遍,不直接复用候选清单
- **避免再回到** Inter、Roboto、Outfit、Plus Jakarta、DM Sans、Space Grotesk、Space Mono、IBM Plex Mono、Fira Code
- 如果要给 hero 单独配 display face,候选:PP Editorial New、PP Neue Machina、TWK Lausanne(此时 General Sans 仍当 body)

## Design Principles

1. **Translucency carries information, not decoration**
   玻璃和 blur 必须服务于「这是浮在上面的层 / 这是沉下去的层」的信息层级。不要把 backdrop-blur 当万能调味剂。如果一个面板拿掉 blur 信息密度不变,就拿掉。

2. **Light is the brand, not the paint**
   紫 → 青 → 品红只用于「光」(光晕、border 高光、focus ring、状态点、生成中的脉冲),不用于「面」(大面积填色、gradient 填字、整块按钮)。Accent 之所以有力量,是因为它稀缺。

3. **Silence between elements**
   深色背景是舞台。生成结果(图片本身)是主角,UI 是观众席。任何 chrome 都不应该比图片本身更亮、更花、更抢戏。空白与暗面不是「没设计」,是「让位」。

4. **Precision is non-negotiable**
   spacing 走 `--d-* / --space-*`,radii 走 `--r-*`,颜色走 token,字号走 type scale。**禁止**用裸 px 数字、裸 hex 色去打补丁。如果 token 不够用,先扩展 token 再用。1-2px 的不一致和未对齐都按 P1 修。

5. **Delight only at moments that earn it**
   动效集中在三个时刻:**任务进入**(start generation)、**结果揭示**(image reveal、history expand)、**完成或失败的反馈**(toast、状态切换)。其他地方安静。不要给静态界面加 idle 动画,不要让光斑无目的地呼吸。

6. **Keyboard-first,但不强迫**
   既然用户是开发者类、长停留,所有主要动作都应该可以纯键盘完成,但视觉上**不要**把这件事炫耀出来(不要到处贴 ⌘K hint)。把它做对、做安静。

## Implementation Notes(给未来设计任务的 hard constraints)

- **不要**引入新的颜色 token、radii token、shadow 档位 — 先用现有的,只在确实不够用时扩展。
- **不要**引入新的 font 家族 — 字体替换是一次性大手术,要做就在专门的任务里做。
- **不要**给整页加 light theme — 深色是核心,light 已退役。
- **不要**用 `border-left/right > 1px` 作为 accent 条 — SKILL 绝对禁令,改用其他结构。
- **不要**写 gradient text — SKILL 绝对禁令,brand gradient 留给「光」。
- **必须**尊重 `prefers-reduced-motion`(`index.css` 已经统一处理,新动效不要绕过)。
- **必须**让所有交互目标在 `pointer: coarse` 下达到 44×44(Tauri 在 macOS 触控板下也会触发)。

## Open Questions(后续设计任务可以决议的)

- 字体是否换、什么时机换?(目前保留 Inter)
- 是否给「生成中」状态加更具材质感的动效(液态光斑、流光描边)?(可在 animate 任务里探索)
- History 缩略图墙是否需要更编辑化的版式(magazine grid)?(可在 layout 任务里探索)
- Settings 是否值得做成「左侧分组导航 + 右侧详情」而不是当前的单列堆叠?(可在 shape 任务里讨论)
